# ☢ АтомДежурство

> Информационная система планирования и учёта дежурств сотрудников объектов атомной отрасли

---

## О проекте

Веб-приложение для автоматизации процессов планирования дежурств персонала на объектах атомной отрасли. Система реализует строгую валидацию назначений смен согласно требованиям безопасности отрасли.

### Ключевые возможности

- 📋 **Учёт персонала** — список сотрудников с уровнями допуска и датами медосмотров
- ⚠️ **Предупреждения** — автоматическая подсветка сотрудников с истекающим медосмотром
- 📅 **Назначение дежурств** — форма назначения смен с мгновенной валидацией
- 🛡️ **4 правила безопасности** — система не позволит назначить смену с нарушением

---

## Правила валидации

При назначении дежурства система последовательно проверяет:

| # | Правило | Описание |
|---|---------|----------|
| 1 | **Активность сотрудника** | Отстранённый сотрудник не может быть назначен на дежурство |
| 2 | **Медосмотр** | Дата смены должна быть раньше даты истечения медосмотра |
| 3 | **Отсутствие конфликтов** | Сотрудник не должен находиться в отпуске / на больничном / в командировке |
| 4 | **Правило 12 часов** | Между сменами должно пройти не менее 12 часов |

## Установка и запуск

### Требования
- Java 17+
- Node.js 18+
- PostgreSQL 14+

### 1. База данных

Создайте базу данных и выполните скрипт:

```sql
CREATE DATABASE nuclear_duty;
```

Затем выполните `01_init_schema.sql` в базе `nuclear_duty`.

### 2. Backend

Настройте подключение в `backend/src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/nuclear_duty
spring.datasource.username=postgres
spring.datasource.password=ВАШ_ПАРОЛЬ
```

Запустите через IntelliJ IDEA или командой:

```bash
cd backend
./mvnw spring-boot:run
```

Backend запустится на `http://localhost:8080`

### 3. Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend запустится на `http://localhost:5173`

### Список персонала
Таблица с подсветкой сотрудников, у которых медосмотр истекает в течение 30 дней.
